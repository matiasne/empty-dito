# Contributing to dito empty local test

Thank you for your interest in contributing! Here are some guidelines to help you get started.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/dito-empty-local-test.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `npm install`

## Development Workflow

1. Make your changes in the appropriate files
2. Test your changes by running the development server: `npm run dev`
3. Ensure your code follows the project's style guide by running:
   - `npm run lint:js` for JavaScript
   - `npm run lint:css` for CSS
   - `npm run format` to auto-format code

## Code Style

- Follow the ESLint and Stylelint configurations
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused
- Use ES6+ features where appropriate

## Commit Messages

- Use clear and descriptive commit messages
- Start with a verb in present tense (e.g., "Add feature", "Fix bug", "Update documentation")
- Keep the first line under 72 characters
- Add more details in the body if necessary

Example:

```
Add user authentication feature

- Implement login form
- Add validation
- Create session management
```

## Pull Requests

1. Ensure your code passes all linting checks
2. Update documentation if you've added new features
3. Add a clear description of the changes in your PR
4. Reference any related issues

## Testing

- Manually test all changes in multiple browsers
- Check responsive behavior on different screen sizes
- Verify that no console errors appear

## Questions?

Feel free to open an issue if you have any questions or need clarification.

Thank you for contributing!
